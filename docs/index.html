CHROMAPLEX 8

"Plug in. Power up. Play." — Fantasy Game Console Documentation v1.0

Overview

Chromaplex 8 is a fantasy game console — a self-contained virtual machine for making, sharing, and playing small retro-style games. It includes a 256×144 widescreen display, 64-color palette, 4-channel synthesizer, and a complete Lua 5.4 scripting environment. Think of it as an imaginary console from a parallel universe where the Super Nintendo had a built-in BASIC prompt.

Games are distributed as cartridges (plain .lua or .cx8 files) that contain code, sprite art, and map data all in one file. The console ships with built-in editors for code, sprites, tile maps, and sound effects — everything you need to create a complete game without leaving the console.

Building from Source

Dependencies

  • GCC — via w64devkit (portable, no install needed)
  • SDL2 2.30+ — MinGW development build from libsdl.org
  • Lua 5.4 — compiled as a static library

Directory Layout

Chromaplex 8/
├── src/              # All source files (*.c, *.h)
├── deps/
│   ├── w64devkit/    # Portable GCC toolchain
│   ├── SDL2-2.30.11/ # SDL2 MinGW build
│   └── lua-5.4.7/    # Lua source (compiled to liblua.a)
├── carts/            # Cartridge files (.lua, .cx8)
├── build/            # Output directory
├── docs/             # This documentation
├── build.bat         # Windows build script
└── build.ps1         # PowerShell build script

Compile Command

deps\w64devkit\bin\gcc.exe -std=c11 -O2 -Wall -Wextra ^
  -I"deps\SDL2-2.30.11\x86_64-w64-mingw32\include\SDL2" ^
  -I"deps\lua-5.4.7\src" -Isrc ^
  src\main.c src\cx8_gpu.c src\cx8_font.c src\cx8_input.c ^
  src\cx8_apu.c src\cx8_memory.c src\cx8_cart.c ^
  src\cx8_modules.c src\cx8_scripting.c src\cx8_home.c ^
  src\cx8_editor.c src\cx8_ed_code.c src\cx8_ed_sprite.c ^
  src\cx8_ed_map.c src\cx8_ed_sfx.c ^
  -L"deps\SDL2-2.30.11\x86_64-w64-mingw32\lib" ^
  -L"deps\lua-5.4.7\src" ^
  -lmingw32 -lSDL2main -lSDL2 -llua -lm ^
  -o build\chromaplex8.exe

Or simply run build.bat / build.ps1.

Copy SDL2.dll from deps\SDL2-2.30.11\x86_64-w64-mingw32\bin\ into the build\ folder next to the exe.

Running

# Launch the home screen (default)
chromaplex8.exe

# Run a specific cartridge directly
chromaplex8.exe carts\hello.lua

# Specify a custom carts folder and window scale
chromaplex8.exe --carts mygames --scale 3

If no cartridge path is given, the console boots to the home screen, where you can browse and launch carts found in the carts/ directory.

Home Screen

The home screen is a PICO-8-style cart browser. It scans the carts directory for .lua and .cx8 files and displays them in a scrollable list. Cart titles and authors are automatically extracted from file headers.

While a game is running, press Esc to return to the home screen.

Cartridge Format

Cartridges are plain text files. A .lua file with a header comment is the simplest format. The .cx8 format embeds sprite and map data after the Lua source.

Minimal Cart

--[[cx8 cart]]
-- title:  My Game
-- author: Your Name
-- desc:   A short description

function _init()
  -- called once at start
end

function _update()
  -- called every frame (60 fps)
end

function _draw()
  cls(0)
  print("HELLO WORLD", 100, 68, 7)
end

.cx8 Embedded Data Sections

When saved from the built-in editor, the cart file includes hex-encoded data sections:

--[[cx8 cart]]
-- title: My Game

-- ... your Lua code above ...

__sprites__
(hex-encoded 128×128 spritesheet data)
__map__
(hex-encoded 256×256 map data)
__end__

Code Editor (F1)

A full-featured text editor with Lua syntax highlighting. Supports up to 4096 lines of code with 256 columns per line.

Syntax Highlighting

  • Keywords — function, end, if, then, else, for, while, do, return, local, nil, true, false, and, or, not, repeat, until, break, in, elseif, goto
  • Built-in functions — All CX8 API functions (cls, spr, btn, etc.)
  • Strings — "text" and 'text'
  • Comments — Lines starting with --
  • Numbers — Numeric literals

Callbacks

Your cartridge can define three special functions that the console calls automatically:

function _init()
Called once when the cartridge starts. Use this to set up your game state, generate sprites, initialise variables.
function _update()
Called every frame (60 times per second) before _draw(). Handle input, move objects, update game logic here.
function _draw()
Called every frame after _update(). All rendering should happen here. Always call cls() first to clear the screen.

Drawing API

cls([col])
Clear the entire screen to color col (default 0, black).
pset(x, y, col)
Set pixel at (x, y) to color col (0–63).
pget(x, y) → col
Get the color index of the pixel at (x, y).
line(x0, y0, x1, y1, col)
Draw a line from (x0,y0) to (x1,y1) in color col.
rect(x0, y0, x1, y1, col)
Draw a rectangle outline from (x0,y0) to (x1,y1).
rectfill(x0, y0, x1, y1, col)
Draw a filled rectangle from (x0,y0) to (x1,y1).
circ(cx, cy, r, col)
Draw a circle outline centered at (cx,cy) with radius r.
circfill(cx, cy, r, col)
Draw a filled circle centered at (cx,cy) with radius r.
tri(x0, y0, x1, y1, x2, y2, col)
Draw a triangle outline with vertices at (x0,y0), (x1,y1), (x2,y2).
trifill(x0, y0, x1, y1, x2, y2, col)
Draw a filled triangle with vertices at (x0,y0), (x1,y1), (x2,y2).
print(str, [x], [y], [col]) → width
Print text str at position (x, y) in color col (default 7 = white). Returns the width in pixels of the printed text. Characters are 4×6 pixels with 1px spacing. Defaults: x=0, y=0, col=7.

Sprite API

spr(n, x, y, [w], [h], [flip_x], [flip_y], [scale], [angle])
Draw sprite n (0–255) at screen position (x, y).
w, h — size in sprite units (default 1×1 = 8×8 pixels). Use 2,2 for a 16×16 region.
flip_x, flip_y — boolean, mirror horizontally/vertically.
scale — float, 1.0 = normal size.
angle — float, rotation in degrees.
sset(x, y, col)
Set a pixel on the spritesheet at (x, y) to color col. The spritesheet is 128×128 pixels.
sget(x, y) → col
Get the color index of a pixel on the spritesheet at (x, y).
Sprite n occupies the 8×8 region starting at x = (n % 16) * 8, y = (n / 16) * 8 on the spritesheet. Sprite 0 is top-left, sprite 15 is top-right, sprite 16 starts the second row.

Map API

map(cel_x, cel_y, sx, sy, cel_w, cel_h)
Draw a region of the tile map to the screen. cel_x, cel_y = top-left cell in map space. sx, sy = screen destination. cel_w, cel_h = number of cells to draw wide/tall. Each cell is rendered as the 8×8 sprite it references.
mset(x, y, tile)
Set map cell at (x, y) to sprite index tile (0–255).
mget(x, y) → tile
Get the sprite index stored in map cell (x, y).

Camera & Clipping

camera([x], [y], [zoom])
Set the camera offset and zoom level. All subsequent drawing is offset by (-x, -y) and scaled by zoom (default 1.0). Call camera() with no arguments to reset.
clip(x, y, w, h)
Set a clipping rectangle. Only pixels inside this region will be drawn. Call clip() with no arguments to reset to full screen.

Palette

pal(c0, c1)
Remap draw color c0 to display as color c1. Affects all subsequent drawing. Useful for palette swaps (e.g., changing a character's outfit color).
pal()
Reset all palette remaps to their defaults.

Input API

btn(id) → bool
Returns true if button id is currently held down.
btnp(id) → bool
Returns true if button id was just pressed this frame (rising edge). Useful for menu navigation and one-shot actions.
Use the BTN_* constants instead of raw numbers: btn(BTN_LEFT), btnp(BTN_A).

Audio API

sfx(channel, freq, [vol], [waveform], [duty])
Start playing a sound on channel (0–3, or 0–15 with SYNTHWAVE-16).
freq — frequency in Hz (e.g., 440 for concert A).
vol — volume 0.0–1.0 (default 0.5).
waveform — WAVE_SQUARE, WAVE_TRIANGLE, WAVE_SAW, WAVE_NOISE, WAVE_PULSE, or WAVE_SINE (default WAVE_SQUARE).
duty — pulse width / duty cycle 0.0–1.0 (default 0.5). Only meaningful for WAVE_PULSE.
envelope(channel, attack, decay, sustain, release)
Set the ADSR envelope for a channel. All times are in seconds.
attack — time to ramp from 0 to full volume.
decay — time to fall from full volume to sustain level.
sustain — volume level during the sustain phase (0.0–1.0).
release — time to fade to 0 after the note ends.
quiet([channel])
Stop a specific channel, or call quiet() with no arguments to silence all channels. 
-- Play a short "coin" sound effect
sfx(0, 880, 0.3, WAVE_SQUARE)
envelope(0, 0.01, 0.1, 0.0, 0.05)

-- Play a bass note on channel 1
sfx(1, 110, 0.6, WAVE_TRIANGLE)
envelope(1, 0.05, 0.2, 0.4, 0.3)

Memory API

Direct access to the console's flat address space (128 KB base RAM).

peek(addr) → val
Read a single byte (0–255) from address addr.
poke(addr, val)
Write byte val (0–255) to address addr.
peek16(addr) → val
Read a 16-bit value (little-endian) from addr.
poke16(addr, val)
Write a 16-bit value to addr.
memcpy(dest, src, len)
Copy len bytes from address src to address dest.
memset(dest, val, len)
Fill len bytes at address dest with value val.

Module API

mod_load(id) → bool
Load expansion module id (0–4). Returns true on success. See the Modules Reference for IDs.
mod_check(id) → bool
Returns true if module id is currently loaded.
mod_info(id) → table | nil
Returns a table with information about the module, or nil if the ID is invalid.
Fields: name, manufacturer, description, flavor, loaded (boolean). 
-- Load extra RAM and check it worked
if mod_load(MOD_TURBO_RAM) then
  print("256 KB RAM UNLOCKED!", 10, 10, 11)
end

-- Query module info
local info = mod_info(MOD_SYNTHWAVE16)
print(info.name)          -- "SYNTHWAVE-16"
print(info.manufacturer)  -- "AudioLux Labs"

Math & System

rnd([max]) → number
Returns a random float in the range [0, max). Default max is 1.0.
flr(x) → number
Floor — rounds x down to nearest integer.
ceil(x) → number
Ceiling — rounds x up to nearest integer.
sin(x) → number
Sine function. Input is 0–1 (not radians!), where 1.0 = a full turn. This matches PICO-8 convention. Returns values in the range [-1, 1].
cos(x) → number
Cosine function. Input is 0–1 (not radians!), where 1.0 = a full turn.
atan2(dx, dy) → number
Returns the angle from the origin to point (dx, dy) as a value in the range [0, 1), where 0 = right, 0.25 = up, 0.5 = left, 0.75 = down.
sgn(x) → number
Returns the sign of x: -1, 0, or 1.
mid(a, b, c) → number
Returns the middle value of the three arguments. Useful for clamping: mid(lo, val, hi).
time() → number
Returns the number of seconds elapsed since the cartridge started running.
sin/cos use turns, not radians! sin(0.25) = -1 (pointing up), not sin(0.25 radians). This is PICO-8 compatible behavior.

Expansion Modules

Chromaplex 8 features a fictional "module bay" with 5 expansion cartridges that can modify the virtual hardware at runtime. Load them with mod_load(id).

IDNameManufacturerEffect

Each module has a unique "flavor" tagline accessible via mod_info(id).flavor.

Example: Complete Cartridge

A simple bouncing ball with sound effects:

--[[cx8 cart]]
-- title:  Bouncy Ball
-- author: Chromaplex Team

local ball = { x = 128, y = 72, dx = 2, dy = 1.5, r = 6 }

function _init()
  -- nothing to set up
end

function _update()
  ball.x = ball.x + ball.dx
  ball.y = ball.y + ball.dy

  -- Bounce off walls
  if ball.x - ball.r < 0 or ball.x + ball.r > SCREEN_W then
    ball.dx = -ball.dx
    sfx(0, 440, 0.3, WAVE_SQUARE)
    envelope(0, 0.01, 0.05, 0.0, 0.02)
  end
  if ball.y - ball.r < 0 or ball.y + ball.r > SCREEN_H then
    ball.dy = -ball.dy
    sfx(0, 330, 0.3, WAVE_TRIANGLE)
    envelope(0, 0.01, 0.05, 0.0, 0.02)
  end
end

function _draw()
  cls(0)
  circfill(ball.x, ball.y, ball.r, 11)
  circ(ball.x, ball.y, ball.r, 7)
  print("BOUNCY BALL", 4, 4, 7)
end

Tips for Making Games

  • Always start _draw() with cls() to clear the previous frame.
  • Use btnp() for menus and one-shot actions; btn() for continuous movement.
  • Define sprites in _init() using sset(), or paint them in the sprite editor.
  • Use camera() for scrolling games — move the camera instead of every object.
  • Use mid(lo, val, hi) to clamp values to a range.
  • The time() function is great for animations: sin(time() * 0.5) gives a smooth oscillation.
  • Remember: sin() and cos() use turns (0–1), not radians.
  • Color 0 is black (transparent in sprites). Use colors 1–63 for visible sprite pixels.
